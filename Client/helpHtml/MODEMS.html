Modems


By default Mud Master assumes a Winsock connection. If you want to use a modem
connection you need to add the word modem to the command line when you run Mud
Master. "mudmast modem"

The first thing you need to do is open a com port to use. You do that with the
/openport command. Openport is formatted like this: /openport {com port}
{speed} {data bits} {parity} {stop bits}. Since your port settings are not
likely to change very often it would probably be easiest if you created either
a macro or an alias to hold these settings.

/alias {open} {/openport com1 57600 8 None 1}

Once open port has successfully been called you have access to your modem.
Mud Master is essentially a terminal program from that point on. You dial just
like you would from any other terminal screen: "atdt <number>" Again, making
yourself an alias would be an easy way to take care of this.

/alias {dial} {atdt 555-1212}

The /disconnect command will hang up the modem. /Zap has been modified a
little. If you are using a modem connection it will send the '^]' character
which should abort a telnet connection.

If your dial in account has any VT100 commands for creating menus and such it
will look really messed up. Mud Master translates only color commands. Also,
navigating menus can be difficult -- mud master works by sending a line of
text at a time, not character by character. Two special characters can be used
in the input line, however. Ctrl+C and Ctrl+]. If you need to break out of a
menu you can just press Ctrl+C and then press enter.
